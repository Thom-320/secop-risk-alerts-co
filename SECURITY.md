# Security

No publiques secretos ni tokens Socrata. Usa `.env` local basado en `.env.example`.

El proyecto no debe usarse para tomar decisiones automaticas sobre personas o entidades.
Las salidas son insumos para revision humana y deben validarse con contexto externo.

Para reportar problemas, abre un issue describiendo:

- componente afectado;
- pasos de reproduccion;
- impacto;
- evidencia tecnica.
