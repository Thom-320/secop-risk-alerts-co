# Evaluacion de alternativas

| Alternativa | Ventaja | Riesgo | Decision |
| --- | --- | --- | --- |
| ContratIA Abierta | Datos masivos, utilidad clara, trazabilidad | Calidad heterogenea de fuentes | Seleccionada |
| Priorizacion vial | Impacto social alto | Menos codigo existente | No seleccionada |
| AgroRiesgo territorial | Buen encuadre publico | Mayor dependencia de fuentes externas | No seleccionada |

Se selecciona ContratIA Abierta porque permite demostrar ingenieria de datos completa:
ETL, PostgreSQL, MongoDB, microservicios, dashboard, pruebas y documentacion.
