# Contest Submission Checklist

## Identidad

- [ ] Repo público.
- [ ] Nombre de producto: ContratIA Abierta.
- [ ] Frase ética visible: prioriza revisión humana con evidencia trazable.
- [ ] README orientado a jurado.

## Demo

- [ ] `make product-pipeline` ejecutado.
- [ ] `make validate-product` con `validation/product_validation.json`.
- [ ] Demo URL pública si está disponible.
- [ ] Video demo de 2 a 3 minutos.
- [ ] Dashboard Streamlit revisado.
- [ ] API read-only para demo pública.

## Documentación

- [ ] `docs/model-card.md`.
- [ ] Data cards en `docs/data-cards/`.
- [ ] `docs/crisp_ml.md`.
- [ ] `docs/ethics-note.md`.
- [ ] `docs/demo-guide.md`.
- [ ] `docs/demo-casebook.md`.
- [ ] `docs/validation-summary.md`.
- [ ] `docs/deployment.md`.
- [ ] `docs/fairness_territorial.md`.

## Validación

- [ ] Validación automática documentada.
- [ ] Estado de validación humana documentado.
- [ ] `docs/human_validation_results.md` actualizado solo con revisiones reales.
- [ ] `docs/usability_results.md` actualizado solo con 5 usuarios reales.

## Concurso

- [ ] Registro en “Usos” si aplica.
- [ ] Captura o enlace del registro.
- [ ] Licencia de código y condiciones de datos visibles.
- [ ] No hay lenguaje acusatorio ni métricas fabricadas.
