# Model Card

- Objetivo: priorizar revisión humana de procesos contractuales.
- Inputs: procesos SECOP II, enriquecimiento opcional de SECOP Integrado, match PAA sujeto a compuerta.
- Output: `priority_score`, `priority_band`, `confidence_score`.
- Uso permitido: priorización y exploración analítica.
- Uso no permitido: afirmar corrupción, sancionar o sustituir control fiscal y jurídico.
